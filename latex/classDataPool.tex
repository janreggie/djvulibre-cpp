\doxysubsection{Detailed Description}
Thread safe data storage. The purpose of \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# is to provide a uniform interface for accessing data from decoding routines running in a multi-\/threaded environment. Depending on the mode of operation it may contain the actual data, may be connected to another \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# or may be mapped to a file. Regardless of the mode, the class returns data in a thread-\/safe way, blocking reading threads if there is no data of interest available. This blocking is especially useful in the networking environment (plugin) when there is a running decoding thread, which wants to start decoding as soon as there is just one byte available blocking if necessary.

Access to data in a \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# may be direct (Using \textbackslash{}\+Ref\{get\+\_\+data\}() function) or sequential (See \textbackslash{}\+Ref\{get\+\_\+stream\}() function).

If the \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# is not connected to anything, that is it contains some real data, this data can be added to it by means of two \textbackslash{}\+Ref\{add\+\_\+data\}() functions. One of them adds data sequentially maintaining the offset of the last block of data added by it. The other can store data anywhere. Thus it\textquotesingle{}s important to realize, that there may be \char`\"{}white
spots\char`\"{} in the data storage.

There is also a way to test if data is available for some given data range (See \textbackslash{}\+Ref\{has\+\_\+data\}()). In addition to this mechanism, there are so-\/called \{{\itshape trigger} callbacks\}, which are called, when there is all data available for a given data range.

Let us consider all modes of operation in details\+:

\textbackslash{}begin\{enumerate\} \textbackslash{}item \{\textbackslash{}bf Not connected \mbox{\hyperlink{classDataPool}{Data\+Pool}}\#\}. In this mode the \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# contains some real data. As mentioned above, it may be added ~\newline
 by means of two functions \textbackslash{}\+Ref\{add\+\_\+data\}() operating independent of each other and allowing to add data sequentially and directly to any place of data storage. It\textquotesingle{}s important to call function \textbackslash{}\+Ref\{set\+\_\+eof\}() after all data has been added.

Functions like \textbackslash{}\+Ref\{get\+\_\+data\}() or \textbackslash{}\+Ref\{get\+\_\+stream\}() can be used to obtain direct or sequential access to the data. As long as \textbackslash{}\+Ref\{is\+\_\+eof\}() is \#\+F\+A\+L\+SE\#, \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# will block every reader, which is trying to read unavailable data until it really becomes available. But as soon as \textbackslash{}\+Ref\{is\+\_\+eof\}() is \#\+T\+R\+UE\#, any attempt to read non-\/existing data will read \#0\# bytes.

Taking into account the fact, that \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# was designed to store Dj\+Vu files, which are in I\+FF formats, it becomes possible to predict the size of the \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# as soon as the first \#32\# bytes have been added. This is invaluable for estimating download progress. See function \textbackslash{}\+Ref\{get\+\_\+length\}() for details. If this estimate fails (which means, that stored data is not in I\+FF format), \textbackslash{}\+Ref\{get\+\_\+length\}() returns \#-\/1\#.

Triggers may be added and removed by means of \textbackslash{}\+Ref\{add\+\_\+trigger\}() and \textbackslash{}\+Ref\{del\+\_\+trigger\}() functions. \textbackslash{}\+Ref\{add\+\_\+trigger\}() takes a data range. As soon as all data in that data range is available, the trigger callback will be called.

All trigger callbacks will be called when \#\+E\+OF\# condition has been set.

\textbackslash{}item \{\textbackslash{}bf \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# connected to another \mbox{\hyperlink{classDataPool}{Data\+Pool}}\#\}. In this \{{\itshape slave\}} mode you can map a given \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# to any offsets range inside another \mbox{\hyperlink{classDataPool}{Data\+Pool}}\#. You can connect the slave \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# even if there is no data in the master \mbox{\hyperlink{classDataPool}{Data\+Pool}}\#. Any \textbackslash{}\+Ref\{get\+\_\+data\}() request will be forwarded to the master \mbox{\hyperlink{classDataPool}{Data\+Pool}}\#, and it will be responsible for blocking readers trying to access unavailable data.

The usage of \textbackslash{}\+Ref\{add\+\_\+data\}() functions is prohibited for connected \#\+Data\+Pool\#s.

The offsets range used to map a slave \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# can be fully specified (both start offset and length are positive numbers) or partially specified (the length is negative). In this mode the slave \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# is assumed to extend up to the end of the master \mbox{\hyperlink{classDataPool}{Data\+Pool}}\#.

Triggers may be used with slave \#\+Data\+Pool\#s as well as with the master ones.

Calling \textbackslash{}\+Ref\{stop\}() function of a slave will stop only the slave (and any other slave connected to it), but not the master.

\textbackslash{}\+Ref\{set\+\_\+eof\}() function is meaningless for slaves. They obtain the \mbox{\hyperlink{classByteStream_a6a752fb635ed48366afad0719be4ef08}{Byte\+Stream\+::\+End\+Of\+File}}\# status from their master.

Depending on the offsets range passed to the constructor, \textbackslash{}\+Ref\{get\+\_\+length\}() returns different values. If the length passed to the constructor was positive, then it is returned by \textbackslash{}\+Ref\{get\+\_\+length\}() all the time. Otherwise the value returned is either \#-\/1\# if master\textquotesingle{}s length is still unknown (it didn\textquotesingle{}t manage to parse I\+FF data yet) or it is calculated as \#masters\+\_\+length-\/slave\+\_\+start\#.

\textbackslash{}item \{\textbackslash{}bf \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# connected to a file\}. This mode is quite similar to the case, when the \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# is connected to another \mbox{\hyperlink{classDataPool}{Data\+Pool}}\#. Similarly, the \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# stores no data inside. It just forwards all \textbackslash{}\+Ref\{get\+\_\+data\}() requests to the underlying source (a file in this case). Thus these requests will never block the reader. But they may return \#0\# if there is no data available at the requested offset.

The usage of \textbackslash{}\+Ref\{add\+\_\+data\}() functions is meaningless and is prohibited.

\textbackslash{}\+Ref\{is\+\_\+eof\}() function always returns \#\+T\+R\+UE\#. Thus \textbackslash{}\+Ref\{set\+\_\+eof\}() us meaningless and does nothing.

\textbackslash{}\+Ref\{get\+\_\+length\}() function always returns the file size.

Calling \textbackslash{}\+Ref\{stop\}() function will stop this \mbox{\hyperlink{classDataPool}{Data\+Pool}}\# and any other slave connected to it.

\mbox{\hyperlink{classDataPool_1_1Trigger}{Trigger}} callbacks passed through \textbackslash{}\+Ref\{add\+\_\+trigger\}() function are called immediately.

This mode is useful to read and decode Dj\+Vu files without reading and storing them in full in memory. \textbackslash{}end\{enumerate\}
