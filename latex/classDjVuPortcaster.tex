\doxysubsection{Detailed Description}
Maintains associations between ports. It monitors the status of all ports (have they been destructed yet?), accepts requests and notifications from them and forwards them to destinations according to internally maintained map of routes.

The caller can modify the route map any way he likes (see \textbackslash{}\+Ref\{add\+\_\+route\}(), \textbackslash{}\+Ref\{del\+\_\+route\}(), \textbackslash{}\+Ref\{copy\+\_\+routes\}(), etc. functions). Any port can be either a sender of a message, an intermediary receiver or a final destination. ~\newline


When a request is sent, the \mbox{\hyperlink{classDjVuPortcaster}{Dj\+Vu\+Portcaster}}\# computes the list of destinations by consulting with the route map. Notifications are only sent to \`{}`alive'\textquotesingle{} ports. A port is alive if it is referenced by a valid \textbackslash{}\+Ref\{\mbox{\hyperlink{classGP}{GP}}\} smartpointer. As a consequence, a port usually becomes alive after running the constructor (since the returned pointer is then assigned to a smartpointer) and is no longer alive when the port is destroyed (because it would not be destroyed if a smartpointer was referencing it).

Destination ports are sorted according to their distance from the source. For example, if port \{\textbackslash{}bf A\} is connected to ports \{\textbackslash{}bf B\} and \{\textbackslash{}bf C\} directly, and port \{\textbackslash{}bf B\} is connected to \{\textbackslash{}bf D\}, then \{\textbackslash{}bf B\} and \{\textbackslash{}bf C\} are assumed to be one hop away from \{\textbackslash{}bf A\}, while \{\textbackslash{}bf D\} is two hops away from \{\textbackslash{}bf A\}.

In some cases the requests and notifications are sent to every possible destination, and the order is not significant (like it is for \textbackslash{}\+Ref\{notify\+\_\+file\+\_\+flags\+\_\+changed\}() request). Others should be sent to the closest destinations first, and only then to the farthest, in case if they have not been processed by the closest. The examples are \textbackslash{}\+Ref\{request\+\_\+data\}(), \textbackslash{}\+Ref\{notify\+\_\+error\}() and \textbackslash{}\+Ref\{notify\+\_\+status\}().

The user is not expected to create the \mbox{\hyperlink{classDjVuPortcaster}{Dj\+Vu\+Portcaster}}\# itself. He should use \textbackslash{}\+Ref\{get\+\_\+portcaster\}() global function instead. ~\newline
