\begin{Indent}\textbf{ Dynamic Arrays}\par
{\em These class implement arrays of objects of any type. Each element is identified by an integer subscript. The valid subscripts range is defined by dynamically adjustable lower-\/ and upper-\/bounds. Besides accessing and setting elements, member functions are provided to insert or delete elements at specified positions.

\mbox{\hyperlink{structClass}{Class}} \textbackslash{}\+Ref\{\mbox{\hyperlink{classGArrayTemplate}{G\+Array\+Template}}\} implements all methods for manipulating arrays of type \#\+T\+Y\+PE\#. You should not however create instances of this class. You should instead use one of the following classes\+: \textbackslash{}begin\{itemize\} \textbackslash{}item \mbox{\hyperlink{structClass}{Class}} \textbackslash{}\+Ref\{G\+Array$<$\+T\+Y\+P\+E$>$\} is the most general class, \textbackslash{}item \mbox{\hyperlink{structClass}{Class}} \textbackslash{}\+Ref\{G\+T\+Array$<$\+T\+Y\+P\+E$>$\} is more efficient, but only works for types that do not require sophisticated constructors or destructors, such as the plain old C types (e.\+g. \#int\# or \#char\# ...). \textbackslash{}item \mbox{\hyperlink{structClass}{Class}} \textbackslash{}\+Ref\{G\+P\+Array$<$\+T\+Y\+P\+E$>$\} implements an array of smart-\/pointers \textbackslash{}\+Ref\{G\+P$<$\+T\+Y\+P\+E$>$\} to objects of type \#\+T\+Y\+PE\#. Using this class reduces the size of the code generated by the template instanciation. \textbackslash{}end\{itemize\}

Another variant of dynamic arrays is implemented in file \textbackslash{}\+Ref\{\mbox{\hyperlink{Arrays_8h_source}{Arrays.\+h}}\}. The main difference is that class \textbackslash{}\+Ref\{\mbox{\hyperlink{classTArray}{T\+Array}}\}, \textbackslash{}\+Ref\{\mbox{\hyperlink{classDArray}{D\+Array}}\} and \textbackslash{}\+Ref\{\mbox{\hyperlink{classDPArray}{D\+P\+Array}}\} implement a copy-\/on-\/demand scheme.
