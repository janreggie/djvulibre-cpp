\doxysubsection*{Arrays.\+h}
\label{_amgrp8bbd7060883340e416d77d890b506763}%
Files \#\char`\"{}\+Arrays.\+h\char`\"{}\# and \#\char`\"{}\+Arrays.\+cpp\char`\"{}\# implement three array template classes. \mbox{\hyperlink{structClass}{Class}} \textbackslash{}\+Ref\{\mbox{\hyperlink{classTArray}{T\+Array}}\} implements an array of objects of trivial types such as \#char\#, \#int\#, \#float\#, etc. It is faster than general implementation for any type done in \textbackslash{}\+Ref\{\mbox{\hyperlink{classDArray}{D\+Array}}\} because it does not cope with element\textquotesingle{}s constructors, destructors and copy operators. Although implemented as a template, which makes it possible to incorrectly use \textbackslash{}\+Ref\{\mbox{\hyperlink{classTArray}{T\+Array}}\} with non-\/trivial classes, it should not be done.

A lot of things is shared by these three arrays. That is why there are more base classes\+: \textbackslash{}begin\{itemize\} \textbackslash{}item \textbackslash{}\+Ref\{\mbox{\hyperlink{classArrayBase}{Array\+Base}}\} defines functions independent of the elements type \textbackslash{}item \textbackslash{}\+Ref\{\mbox{\hyperlink{classArrayBaseT}{Array\+BaseT}}\} template class defining functions shared by \textbackslash{}\+Ref\{\mbox{\hyperlink{classDArray}{D\+Array}}\} and \textbackslash{}\+Ref\{\mbox{\hyperlink{classTArray}{T\+Array}}\} \textbackslash{}end\{itemize\}

The main difference between \textbackslash{}\+Ref\{\mbox{\hyperlink{classGArray}{G\+Array}}\} (now obsolete) and these ones is the copy-\/on-\/demand strategy, which allows you to copy array objects without copying the real data. It\textquotesingle{}s the same thing, which has been implemented in \textbackslash{}\+Ref\{G\+String\} long ago\+: as long as you don\textquotesingle{}t try to modify the underlying data, it may be shared between several copies of array objects. As soon as you attempt to make any changes, a private copy is created automatically and transparently for you -\/ the procedure, that we call \char`\"{}copy-\/on-\/demand\char`\"{}.

Also, please note that now there is no separate class, which does fast sorting. Both \textbackslash{}\+Ref\{\mbox{\hyperlink{classTArray}{T\+Array}}\} (dynamic array for trivial types) and \textbackslash{}\+Ref\{\mbox{\hyperlink{classDArray}{D\+Array}}\} (dynamic array for arbitrary types) can sort their elements.

\{\textbackslash{}bf Historical comments\} --- Leon chose to implement his own arrays because the S\+TL classes were not universally available and the compilers were rarely able to deal with such a template galore. Later it became clear that there is no really good reason why arrays should be derived from containers. It was also suggested to create separate arrays implementation for simple classes and do the copy-\/on-\/demand strategy, which would allow to assign array objects without immediate copying of their elements.

At this point \textbackslash{}\+Ref\{\mbox{\hyperlink{classDArray}{D\+Array}}\} and \textbackslash{}\+Ref\{\mbox{\hyperlink{classTArray}{T\+Array}}\} should only be used when it is critical to have the copy-\/on-\/demand feature. The \textbackslash{}\+Ref\{\mbox{\hyperlink{classGArray}{G\+Array}}\} implementation is a lot more efficient.

\doxysubsection{Detailed Description}
Dynamic array base class. This is an auxiliary base class for \textbackslash{}\+Ref\{\mbox{\hyperlink{classDArray}{D\+Array}}\} and \textbackslash{}\+Ref\{\mbox{\hyperlink{classTArray}{T\+Array}}\} implementing some shared functions independent of the type of array elements. It\textquotesingle{}s not supposed to be constructed by hands. Use \textbackslash{}\+Ref\{\mbox{\hyperlink{classDArray}{D\+Array}}\} and \textbackslash{}\+Ref\{\mbox{\hyperlink{classTArray}{T\+Array}}\} instead.
