\doxysubsection{Detailed Description}
\mbox{\hyperlink{classDjVuImage}{Dj\+Vu\+Image}} to Post\+Script converter. Use this class to print \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVuImage}{Dj\+Vu\+Image}}\}s and \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVuDocument}{Dj\+Vu\+Document}}\}s. The behavior is customizable. See \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVuToPS_1_1Options}{Dj\+Vu\+To\+P\+S\+::\+Options}}\} for the description of available options.

Outputs the specifies pages from the \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVuDocument}{Dj\+Vu\+Document}}\} into the \textbackslash{}\+Ref\{\mbox{\hyperlink{classByteStream}{Byte\+Stream}}\} in Post\+Script format. The function will generate a multipage Post\+Script document conforming to PS D\+SC 3.\+0 by storing into it every page mentioned in the \#page\+\_\+range\#.

If \#page\+\_\+range\# is empty, all pages from the \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVuDocument}{Dj\+Vu\+Document}}\} \#doc\# will be printed. The \#page\+\_\+range\# is a set of ranges separated by commas. Every range has this form\+: \{{\itshape start\+\_\+page\}}\mbox{[}-\/\{{\itshape end\+\_\+page\}}\mbox{]}. \{{\itshape end\+\_\+page\}} is optional and can be less than the \{{\itshape start\+\_\+page\}}, in which case the pages will be printed in the reverse order.

Examples\+: \textbackslash{}begin\{itemize\} \textbackslash{}item \{\textbackslash{}bf 1-\/10\} -\/ Will print pages 1 to 10. \textbackslash{}item \{\textbackslash{}bf 10-\/1\} -\/ Will print pages 1 to 10 in reverse order. \textbackslash{}item \{\textbackslash{}bf 1-\/10,12-\/20\} -\/ Will print pages 1 to 20 with page 11 skipped. \textbackslash{}end\{itemize\} \mbox{\Hypertarget{classDjVuToPS_a2b2a4866689380c6867504b65ee1d7bb}\label{classDjVuToPS_a2b2a4866689380c6867504b65ee1d7bb}}

{\footnotesize\ttfamily void Dj\+Vu\+To\+P\+S\+::print (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{classByteStream}{Byte\+Stream}} \&}]{str,  }\item[{\mbox{\hyperlink{classGP}{GP}}$<$ \mbox{\hyperlink{classDjVuImage}{Dj\+Vu\+Image}} $>$}]{dimg,  }\item[{const \mbox{\hyperlink{classGRect}{G\+Rect}} \&}]{prn\+\_\+rect,  }\item[{const \mbox{\hyperlink{classGRect}{G\+Rect}} \&}]{img\+\_\+rect,  }\item[{int}]{override\+\_\+dpi = {\ttfamily -\/1} }\end{DoxyParamCaption})}

Prints the specified \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVuImage}{Dj\+Vu\+Image}}\} \#dimg\# into the \textbackslash{}\+Ref\{\mbox{\hyperlink{classByteStream}{Byte\+Stream}}\} \#str\#. The function will first scale the image to fit the \#img\+\_\+rect\#, then extract \#prn\+\_\+rect\# from the obtained bitmap and will output it in the Post\+Script format. The function generates a legal Post\+Script (or Encapsulated Post\+Script) file taking care of all comments conforming to Document Structure Conventions v. 3.\+0.

\{\textbackslash{}bf Warning\+:\} The zoom factor specified in \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVuToPS_1_1Options}{Options}}\} does not affect the amount of data stored into the Post\+Script file. It will be used by the Post\+Script code to additionally scale the image. We cannot pre-\/scale it here, because we do not know the future resolution of the printer. The \#img\+\_\+rect\# and \#prn\+\_\+rect\# alone define how much data will be sent to printer.

Using \#img\+\_\+rect\# one can upsample or downsample the image prior to sending it to the printer.
