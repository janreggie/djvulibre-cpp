\begin{Indent}\textbf{ G\+Map\+Areas.\+h}\par
{\em Files \#\char`\"{}\+G\+Map\+Areas.\+h\char`\"{}\# and \#\char`\"{}\+G\+Map\+Areas.\+cpp\char`\"{}\# implement base objects used by the plugin to display and manage hyperlinks and highlighted areas inside a \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVuImage}{Dj\+Vu\+Image}}\} page.

The currently supported areas can be rectangular (\textbackslash{}\+Ref\{\mbox{\hyperlink{classGMapRect}{G\+Map\+Rect}}\}), elliptical (\textbackslash{}\+Ref\{\mbox{\hyperlink{classGMapOval}{G\+Map\+Oval}}\}) and polygonal (\textbackslash{}\+Ref\{\mbox{\hyperlink{classGMapPoly}{G\+Map\+Poly}}\}). Every map area besides the definition of its shape contains information about display style and optional \{\textbackslash{}bf U\+RL\}, which it may refer to. If this \{\textbackslash{}bf U\+RL\} is not empty then the map area will work like a hyperlink.

The classes also implement some useful functions to ease geometry manipulations

\doxysubsection{Detailed Description}
Implements polygonal map areas. The only supported types of border are \#\+N\+O\+\_\+\+B\+O\+R\+D\+ER\#, \#\+X\+O\+R\+\_\+\+B\+O\+R\+D\+ER\# and \#\+S\+O\+L\+I\+D\+\_\+\+B\+O\+R\+D\+ER\#. Its contents can not be highlighted either. It\textquotesingle{}s worth mentioning here that despite its name the polygon may be open, which basically makes it a broken line. This very specific mode is used by the hyperlink editor when creating the polygonal hyperlink.
