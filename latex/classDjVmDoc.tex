\doxysubsection{Detailed Description}
Read/\+Write Dj\+Vu multipage documents.

The \char`\"{}new\char`\"{} Dj\+Vu multipage documents can be of two types\+: \{{\itshape bundled\}} and \{{\itshape indirect\}}. In the first case all pages are packed into one file, which is very like an archive internally. In the second case every page is stored in a separate file. Plus there can be other components, included into one or more pages, which also go into separate files. In addition to pages and components, in the case of the \{{\itshape indirect\}} format there is one more top-\/level file with the document directory (see \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVmDir}{Dj\+Vm\+Dir}}\}), which is basically an index file containing the list of all files composing the document.

This class can read documents of both formats and can save them under any format. It is therefore ideal for converting between \{{\itshape bundled\}} and \{{\itshape indirect\}} formats. It cannot be used however for reading obsolete formats. The best way to convert obsolete formats consists in reading them with class \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVuDocument}{Dj\+Vu\+Document}}\} class and saving them using \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVuDocument_a83d0b0e54f455500f8b88774591a359e}{Dj\+Vu\+Document\+::write}}\} or \textbackslash{}\+Ref\{\mbox{\hyperlink{classDjVuDocument_ab1f30abb986d0d2aa1f008a2e4f1514d}{Dj\+Vu\+Document\+::expand}}\}.

This class can also be used to create and modify multipage documents at the low level without decoding every page or component (See \textbackslash{}\+Ref\{insert\+\_\+file\}() and \textbackslash{}\+Ref\{delete\+\_\+file\}()).
