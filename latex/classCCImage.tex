\begin{Indent}\textbf{ cjb2}\par
{\em \{\textbackslash{}bf Synopsis\} \textbackslash{}begin\{verbatim\} cjb2 \mbox{[}options\mbox{]} $<$input-\/pbm-\/or-\/tiff$>$ $<$output-\/djvu$>$ \textbackslash{}end\{verbatim\}

\{\textbackslash{}bf Description\}

File \#\char`\"{}cjb2.\+cpp\char`\"{}\# demonstrates a simple encoder for Bilevel Dj\+Vu Images. It is able to perform lossless encoding and limited lossy encoding. Lots of lossy encoding refinements are missing from this simple implementation. \mbox{\hyperlink{classComments}{Comments}} in the code suggest a few improvements.

\mbox{\hyperlink{structOptions}{Options}} are\+: \textbackslash{}begin\{description\} \textbackslash{}item\mbox{[}-\/dpi xxx\mbox{]} Specify image resolution (default 300). \textbackslash{}item\mbox{[}-\/lossless\mbox{]} Lossless compression (same as -\/losslevel 0, default). \textbackslash{}item\mbox{[}-\/clean\mbox{]} Quasi-\/lossless compression (same as -\/losslevel 1). \textbackslash{}item\mbox{[}-\/lossy\mbox{]} Lossy compression (same as -\/losslevel 100). \textbackslash{}item\mbox{[}-\/losslevel n\mbox{]} Set loss level (0 to 200) \textbackslash{}item\mbox{[}-\/verbose\mbox{]} Display additional messages. \textbackslash{}end\{description\} Encoding is lossless unless one or several lossy options are selected. The \#dpi\# argument mostly affects the cleaning thresholds.

\{\textbackslash{}bf Bugs\}

This is not the full-\/fledged multipage Dj\+Vu compressor, but merely a free tool provided with the Dj\+Vu Reference Library as a demonstrative example.

\begin{Indent}\textbf{ cpaldjvu}\par
{\em \{\textbackslash{}bf Synopsis\} \textbackslash{}begin\{verbatim\} cpaldjvu \mbox{[}options\mbox{]} $<$inputppmfile$>$ $<$outputdjvufile$>$ \textbackslash{}end\{verbatim\}

\{\textbackslash{}bf Description\}

File \#\char`\"{}cpaldjvu.\+cpp\char`\"{}\# demonstrates a simple quasi-\/lossless compressor for low resolution, low color, images with a reduced number of colors (e.\+g screendumps). It simply quantizes the image on a limited number of colors, uses the dominant color to construct a uniform background, then performs lossless jb2 compression for all remaining objects. ~\newline


\mbox{\hyperlink{structOptions}{Options}} \textbackslash{}begin\{description\} \textbackslash{}item\mbox{[}-\/colors n\mbox{]} Maximum number of colors during quantization (default 256) \textbackslash{}item\mbox{[}-\/dpi n\mbox{]} Resolution written into the output file (default 100). \textbackslash{}item\mbox{[}-\/verbose\mbox{]} Displays additional messages. \textbackslash{}end\{description\}

\{\textbackslash{}bf Remarks\}

This is an interesting alternative to G\+IF. It performs especially well on screendumps. Compression ratios can get hurt when there are continuous tone segment in the image. Demoting such segments from foreground to background is a pretty interesting project. Dithered segments behave surprisingly well.
