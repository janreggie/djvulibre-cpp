\doxysubsection{Detailed Description}
Intuitive interface to I\+FF files.

It\textquotesingle{}s too terrible to keep reading/writing I\+FF files chunk after chunk using \{\textbackslash{}\+Ref \mbox{\hyperlink{classIFFByteStream}{I\+F\+F\+Byte\+Stream}}\}s. This class allows you to operate with chunks as with structures or arrays without even caring about the byte streams.

Some of the examples are below\+: \textbackslash{}begin\{verbatim\} \mbox{\hyperlink{classGP}{G\+P$<$\+G\+I\+F\+F\+Chunk$>$}} chunk; chunk=manager1.\+get\+\_\+chunk(\char`\"{}\+B\+G44\mbox{[}2\mbox{]}\char`\"{}); manager2.\+add\+\_\+chunk(\char`\"{}.\+F\+O\+R\+M\+:\+D\+J\+V\+U.\+B\+G44\mbox{[}-\/1\mbox{]}\char`\"{}, chunk); \textbackslash{}end\{verbatim\}

\{\textbackslash{}bf Chunk name\} \textbackslash{}begin\{itemize\} \textbackslash{}item Every chunk name may contain optional prefix \#\+F\+O\+RM\+:\#, \#\+L\+I\+ST\+:\#, \#\+P\+R\+OP\+:\# or \#\+C\+AT\+:\#. If the prefix is omitted and the chunk happens to contain other chunks, \#\+F\+O\+RM\+:\# will be assumed. \textbackslash{}item Every chunk name may be \{{\itshape short\}} or \{{\itshape complete\}}. \{{\itshape short\}} chunk names may not contain dots as they\textquotesingle{}re a subchunks names with respect to a given chunk. \{{\itshape complete\}} chunk names may contain dots. But there may be or may not be the \{{\itshape leading} dot\} in the name. If the \{{\itshape leading} dot\} is present, then the name is assumed to contain the name of the top-\/level chunk as well. Otherwise it\textquotesingle{}s treated \{{\itshape with} respect\} to the top-\/level chunk. You may want to use the leading dot only when you add a chunk to an empty document, since a command like \#manager.\+add\+Chunk(\char`\"{}.\+F\+O\+R\+M\+:\+D\+J\+V\+U.\+B\+G44\char`\"{}, chunk)\# will create the top level chunk of the requested type (\#\+F\+O\+RM\+:D\+J\+VU\#) and will add chunk \#\+B\+G44\# to it \{{\itshape automatically\}}. \textbackslash{}item You may use \{{\itshape brackets\}} in the name to specify the chunk\textquotesingle{}s position. The meaning of the number inside the brackets depends on the function you call. In most of the cases this is the number of the chunk with the given name in the parent chunk. But sometimes (as in \#add\+Chunk(name, buffer, length)\#) the brackets at the end of the \#name\# actually specify the \{{\itshape position\}} of the chunk in the parent. For example, to insert \#\+I\+N\+CL\# chunk into \#\+D\+J\+VU\# form at position \#1\# (make it the second) you may want to use \#manager.\+add\+Chunk(\char`\"{}.\+D\+J\+V\+U.\+I\+N\+C\+L\mbox{[}1\mbox{]}\char`\"{}, data, size)\#. At the same time, to get 2-\/nd chunk with name \#\+B\+G44\# from form \#\+D\+J\+VU\# you should do smth like \#chunk=manager.\+get\+Chunk(\char`\"{}\+B\+G44\mbox{[}1\mbox{]}\char`\"{})\#. Note, that here the manager will search for chunk \#\+B\+G44\# in form \#\+D\+J\+VU\# and will take the second \{{\itshape found\}} one. \textbackslash{}end\{itemize\}
